## 两个 Bug 根因排查

### Bug 1: KMStore.ToolItem 缺失 'healthCheck' 成员编译错误
- **结论**: 已修复。`ToolItem` 枚举已包含 `case healthCheck = "healthCheck"` (AppStore.swift:53)
- **来源**: MEMORY.md 中记录了此问题，但最新构建日志显示 BUILD SUCCEEDED
- **KMStore**: 此名称不存在于代码库中，已重命名为 `KnowledgeStore`，`ToolItem` 实际属于 `AppStore`

### Bug 2: iPad 性能监控卡片无法弹出 sheet 的根因
- **根因 1 (致命)**: `showPerfDashboard` 无任何 UI 触发入口 — 全代码库中没有将其设为 `true` 的代码
- **根因 2 (致命)**: `AppStore.settingsStore` 被标记为 `@ObservationIgnored`，且 `AppStore.showPerfDashboard` 计算属性缺少 `access(keyPath:)` / `withMutation(keyPath:)` 调用，导致 SwiftUI `@Bindable` 绑定无法感知变更
- **根因 3 (次要)**: `SettingsStore.showPerfDashboard` 直接用 `public var` 声明而非 `@ObservationIgnored` + `access/withMutation` 模式，虽然自身可观察，但经 AppStore 转发时断裂
- **对比**: `isPrivacyModeEnabled` 在 SettingsStore 中使用了正确的 `access/withMutation` 模式，但 AppStore 的转发也未加追踪
- **sheet 挂载位置**: `AppLayoutComponents.swift` 中三处（adaptiveSplitView:122, modernTabView:171, legacyTabView:239）
