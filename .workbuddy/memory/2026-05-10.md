## 修复 iPhone 17 Pro 模拟器编译错误

**问题根因**：`SynthesisDocRow.swift` 的 `#else`（iOS/macOS）分支声明了 `let editMode: EditMode`，但没有定义 `EditMode` 枚举，编译器误将其解析为 SwiftUI 内置 `EditMode`，导致与 `SynthesisView.AppEditMode` 类型不匹配，报错：
> `cannot convert value of type 'SynthesisView.AppEditMode' to expected argument type 'EditMode'`（SynthesisView.swift:202）

**修复方案**（两步）：
1. `SynthesisDocRow.swift`：在 `#else` 分支补充 `enum EditMode { case active, inactive }` 定义。
2. `SynthesisView.swift`：将 `enum AppEditMode` 声明改为 `#if os(watchOS)` 条件编译，iOS 分支直接使用 `SynthesisDocRow.EditMode` 类型。

**结果**：`xcodebuild` 在 `platform=iOS Simulator,name=iPhone 17 Pro` 目标上编译成功（BUILD SUCCEEDED）。

## 产品设计方案：笔记本层 + 个人中心 + 登录注册

**背景**：当前应用缺少笔记本维度（所有页面平铺）、登录/注册入口、以及个人设置快捷入口（Settings 作为独立 Tab 与内容同级，不符合 NotebookLM 范式）。

**三大变更方向**：
1. **登录/注册前置层**：App 启动时检查 AuthSession，未登录展示 Login/Register 页（邮箱+密码、Sign in with Apple、访客模式三条路径）。
2. **Notebook Hub**：登录后首页为笔记本工作台，类似 NotebookLM，以 2 列卡片展示所有笔记本；点击后进入笔记本上下文，当前所有 Tab（Wiki/导入/图谱/搜索）均属该笔记本作用域。
3. **个人中心**：移除 Settings Tab，右上角头像点击展开 Sheet，内含用户信息+统计卡片+原 Settings 内容（账户安全、隐私数据、外观主题、AI 模型设置、通用设置、退出登录）。

**代码影响文件**：`ContentView.swift`（启动路由）、`AppTab.swift`（移除 settings case）、`SidebarView.swift`（Hub 中显示笔记本列表）、`NavigationView.swift`（头像 ToolbarItem）。
