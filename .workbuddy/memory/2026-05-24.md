## macOS 编译错误根因排查

### 根因：MacOSPlatformCapabilities.swift 缺少条件编译保护

**错误信息**：
- `MacOSPlatformCapabilities.swift:51:55: error: 'withSecurityScope' is unavailable in iOS`
- `MacOSPlatformCapabilities.swift:64:70: error: 'withSecurityScope' is unavailable in iOS`

**根因**：
1. `MacOSSecurityScopedStorage` 使用了 `.withSecurityScope` (macOS AppKit 专属 API)，但该文件**没有 `#if os(macOS)` 条件编译保护**
2. 同目录其他文件（MacAccessibilityService、MacFileArchiver、MacPasteboardService）都有正确的条件编译保护
3. `MacOSBiometricAuthProvider` 也在同一文件中，但 LAContext API 跨平台可用，所以不报错

**触发场景**：
- 用户在 Xcode 中选择 ZhiYuMac scheme + **iPhone Simulator** destination
- 这导致 `os(iOS)` = true，iOS SDK 中不存在 `.withSecurityScope`
- 而 Catalyst 构建 (`Debug-maccatalyst`) 中 `os(macOS)` = true，`withSecurityScope` 在 macOS SDK 下可用，所以命令行 Catalyst 构建成功

**对比**：
| 文件 | 条件编译保护 | macOS 专属 API | iOS 编译 |
|------|------------|--------------|---------|
| MacAccessibilityService.swift | `#if os(macOS) && !targetEnvironment(macCatalyst)` | NSAccessibility | ✅ 跳过 |
| MacFileArchiver.swift | `#if os(macOS) && !targetEnvironment(macCatalyst)` | Process() | ✅ 跳过 |
| MacPasteboardService.swift | `#if os(macOS)` | NSPasteboard | ✅ 跳过 |
| MacOSPlatformCapabilities.swift | **无保护** | `.withSecurityScope` | ❌ 报错 |

**修复方案**：给 `MacOSSecurityScopedStorage` 添加 `#if os(macOS)` 保护，并为 Catalyst/iOS 提供降级实现

### 修复已执行 ✅

**修改文件 1**：`Sources/Platforms/macOS/MacOSPlatformCapabilities.swift`
- 给 `MacOSSecurityScopedStorage` 添加 `#if os(macOS)` / `#endif` 包裹
- iOS Simulator 下该类型不编译，`ModuleRegistrar` 中 `#if os(macOS)` 注册代码也不执行，fallback 到 `iOSSecurityScopedStorage`

**修改文件 2**：`Sources/App/ModuleRegistrar.swift`
- `MacFileArchiver` 注册：`#if os(macOS)` → `#if os(macOS) && !targetEnvironment(macCatalyst)`
- `MacAccessibilityService` 注册：`#if os(macOS)` → `#if os(macOS) && !targetEnvironment(macCatalyst)`
- Catalyst 下 fallback 到 `iOSFileArchiver` / `iOSAccessibilityService`

**验证结果**：
- ZhiYuMac + iPad Simulator: BUILD SUCCEEDED ✅
- ZhiYuMac + Mac Catalyst: BUILD SUCCEEDED ✅
- ZhiYu + iOS device: BUILD SUCCEEDED ✅
